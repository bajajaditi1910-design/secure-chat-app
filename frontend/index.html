About the Project

This secure chat application implements military-grade end-to-end encryption using Python sockets and the cryptography library. The system ensures complete message privacy through a blind relay server architecture where the server cannot decrypt or read any messages.

Built with a focus on security and privacy, the application demonstrates fundamental cryptographic principles and secure communication protocols in a practical, real-world implementation.

Security Features

Industry-standard cryptographic protocols

Diffie-Hellman Key Exchange

Secure key establishment protocol that allows two parties to establish a shared secret over an insecure channel. Each client generates ephemeral key pairs for perfect forward secrecy.

AES Encryption

Advanced Encryption Standard (AES) in CBC mode with PKCS7 padding. All messages are encrypted with 256-bit keys, providing military-grade protection for your conversations.

HMAC Integrity Verification

Hash-based Message Authentication Code (HMAC) using SHA-256 ensures message integrity and authenticity, protecting against tampering and man-in-the-middle attacks.

Zero-Trust / Blind Relay Server

The relay server has zero knowledge of message contents. All encryption and decryption happens on client devices, ensuring true end-to-end encryption with no server-side access.

Encryption Flow

Step-by-step message encryption process

1

Key Exchange Initialization

Both clients generate their Diffie-Hellman key pairs (private and public keys) using secure random number generation.

2

Public Key Exchange

Clients exchange public keys through the relay server without exposing their private keys.

3

Shared Secret Derivation

Each client computes the shared secret independently using their private key and the other party's public key.

4

Session Key Generation

The shared secret is used to derive AES encryption keys and HMAC keys through a key derivation function.

5

Message Encryption

Messages are encrypted using AES-256 in CBC mode with a random initialization vector (IV) for each message.

6

HMAC Generation

An HMAC is computed over the encrypted message to ensure integrity and authenticity.

7

Relay Transmission

The encrypted message and HMAC are sent through the blind relay server which cannot decrypt the contents.

8

Verification & Decryption

The recipient verifies the HMAC, then decrypts the message using the shared session key.
