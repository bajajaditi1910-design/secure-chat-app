Secure End-to-End Encrypted Chat Application

A Python-based secure messaging system featuring Diffie-Hellman key exchange, AES-128 encryption, HMAC integrity verification, and zero-trust blind relay architecture.

Security Features

Diffie-Hellman Key Exchange

Establishes shared secret between clients without transmitting the key over the network.

AES-128 Encryption

Military-grade symmetric encryption with CBC mode and PKCS7 padding for message confidentiality.

HMAC Integrity

SHA-256 based message authentication code ensures messages haven't been tampered with.

Zero-Trust Server

Blind relay server cannot decrypt messages. True end-to-end encryption with client-side keys.

Encryption Flow

1

Connection Establishment

Both clients connect to the relay server via TCP socket on port 5000.

2

Ready Signal

Server sends "READY" signal when two or more clients are connected.

3

Key Generation

Each client generates a Diffie-Hellman private/public key pair using 2048-bit parameters.

4

Public Key Exchange

Clients exchange public keys through the server. Initiator sends first, responder replies.

5

Shared Secret Computation

Both clients compute the same shared secret using their private key and peer's public key.

6

Key Derivation

SHA-256 hash of shared secret generates master key. Split into AES key (16 bytes) and HMAC key (16 bytes).

7

Message Encryption

Messages encrypted with AES-128-CBC, random IV per message. HMAC-SHA256 appended for integrity.

8

Secure Communication

Encrypted messages relayed through server. Only intended recipient can decrypt and verify.
